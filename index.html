Introducción

GitHub Pages nos permite subir archivos HTML, CSS y Javascript como si fuera un repositorio normal de Git y nos permite ejecutar páginas webs del lado del cliente.
GitHub Pages no funciona para tecnologías de Servidor ya que no se encarga de compilar archivos ni de interpretar.

Funciona para tener documentación, páginas personales, páginas para algún proyecto en específico, etc.

Primeros pasos

  • Código

    Tener en una carpeta local todo el código que va a ser subido a git para poder mostrarlo a GitHub Pages.

  • Repositorio desde GitHub

    Podemos crear un repositorio dónde vamos a subir todos nuestros archivos que queremos que se muestre.

  • Git Init

    Iniciar con Git Init desde la terminal donde está ubicada nuestra carpeta con todos los archivos a subir.

Segundo Paso

                
                    cd nombreDeLaCarpeta
                    git init
                    git add .
                    git commit -m "Mensaje"
                    
                    //Ingresar a github.com y crear un repositorio
                    
                    git remote add origin https://github.com/usuario/nombreDeRepositorio.git
                    git push -u origin master

Subimos los commits creados a la rama master de Github.

                
                    //Creamos un rama nueva
                    
                    git branch gh-pages
                    git checkout gh-pages
                    git branch
                    git push origin gh-pages
